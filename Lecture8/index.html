Bias–Variance Tradeoff

Consider the general regression setup where we are given a random pair (X,Y)∈Rp×R(X, Y) \in \mathbb{R}^p \times \mathbb{R}(X,Y)∈Rp×R. We would like to "predict" YYY with some function of XXX, say, g(X)g(X)g(X).

To clarify what we mean by "predict," we specify that we would like g(X)g(X)g(X) to be "close" to YYY (which is, by definition, given by Y=f(X)+ϵY = f(X) + \epsilonY=f(X)+ϵ).

To further clarify what we mean by "close," we define the squared error loss of estimating YYY using g(X)g(X)g(X).

L(Y,g(X))≜(Y−g(X))2L(Y, g(X)) \triangleq (Y - g(X)) ^ 2L(Y,g(X))≜(Y−g(X))2

Now we can clarify the goal of regression, which is to minimize the above loss, on average. We call this the risk of estimating YYY using g(X)g(X)g(X).

R(Y,f(X))≜E[L(Y,g(X))]=EX,Y[(Y−g(X))2]R(Y, f(X)) \triangleq \mathbb{E}[L(Y, g(X))] = \mathbb{E}_{X, Y}[(Y - g(X)) ^ 2]R(Y,f(X))≜E[L(Y,g(X))]=EX,Y​[(Y−g(X))2]

Before attempting to minimize the risk, we first re-write the risk after conditioning on XXX.

EX,Y[(Y−g(X))2]=EXEY∣X[(Y−g(X))2∣X=x]\mathbb{E}_{X, Y} \left[ (Y - g(X)) ^ 2 \right] = \mathbb{E}_{X} \mathbb{E}_{Y \mid X} \left[ ( Y - g(X) ) ^ 2 \mid X = x \right]EX,Y​[(Y−g(X))2]=EX​EY∣X​[(Y−g(X))2∣X=x]

Minimizing the right-hand side is much easier, as it simply amounts to minimizing the inner expectation with respect to Y∣XY \mid XY∣X. That is, we need to (essentially) minimize the risk pointwise for each xxx.

It turns out that the risk is minimized by the conditional mean of YYY given XXX,

g(x)=E(Y∣X=x)g(x) = \mathbb{E}(Y \mid X = x)g(x)=E(Y∣X=x)

This ggg is often called the regression function; we'll see why in a moment.

Note that the choice of squared error loss is somewhat arbitrary. Suppose instead we chose absolute error loss.

L(Y,f(X))≜∣Y−g(X)∣L(Y, f(X)) \triangleq | Y - g(X) |L(Y,f(X))≜∣Y−g(X)∣

The risk would then be minimized by the conditional median.

g(x)=median(Y∣X=x)g(x) = \text{median}(Y \mid X = x)g(x)=median(Y∣X=x)

Despite this possibility, our preference will still be for squared error loss. The reasons for this are numerous, including: historical reasons, ease of optimization, large-sample properties, and protecting against large deviations.

Now, given data D=(xi,yi)∈Rp×R\mathcal{D} = (x_i, y_i) \in \mathbb{R}^p \times \mathbb{R}D=(xi​,yi​)∈Rp×R, our goal becomes finding some g^\hat{g}g^​ that is a good estimate of the regression function ggg. We'll see that this amounts to minimizing what we call the reducible error.

Reducible and Irreducible Error

Suppose that we obtain some g^\hat{g}g^​. How well does it estimate ggg? We define the expected prediction error of predicting YYY using g^(X)\hat{g}(X)g^​(X). A good g^\hat{g}g^​ will have a low expected prediction error.

EPE(Y,g^(X))≜EX,Y,D[(Y−g^(X))2]\text{EPE}\left(Y, \hat{g}(X)\right) \triangleq \mathbb{E}_{X, Y, \mathcal{D}} \left[ \left( Y - \hat{g}(X) \right)^2 \right]EPE(Y,g^​(X))≜EX,Y,D​[(Y−g^​(X))2]

This expectation is over XXX, YYY, and also D\mathcal{D}D. The estimate g^\hat{g}g^​ is actually random depending on the sampled data D\mathcal{D}D. We could actually write g^(X,D)\hat{g}(X, \mathcal{D})g^​(X,D) to make this dependence explicit, but our notation will become cumbersome enough as it is.

Like before, we'll condition on XXX. This results in the expected prediction error of predicting YYY using g^(X)\hat{g}(X)g^​(X) when X=xX = xX=x.

EPE(Y,g^(x))=EY∣X,D[(Y−g^(X))2∣X=x]=ED[(f(x)−g^(x))2]⏟reducible error+VY∣X[Y∣X=x]⏟irreducible error\text{EPE}\left(Y, \hat{g}(x)\right) = \mathbb{E}_{Y \mid X, \mathcal{D}} \left[ \left(Y - \hat{g}(X) \right)^2 \mid X = x \right] = \underbrace{\mathbb{E}_{\mathcal{D}} \left[ \left(f(x) - \hat{g}(x) \right)^2 \right]}_\textrm{reducible error} + \underbrace{\mathbb{V}_{Y \mid X} \left[ Y \mid X = x \right]}_\textrm{irreducible error}EPE(Y,g^​(x))=EY∣X,D​[(Y−g^​(X))2∣X=x]=reducible errorED​[(f(x)−g^​(x))2]​​+irreducible errorVY∣X​[Y∣X=x]​​

A number of things to note here:

  • The expected prediction error is for a random YYY given a fixed xxx and a random f^\hat{f}f^​. As such, the expectation is over Y∣XY \mid XY∣X and D\mathcal{D}D. Our estimated function f^\hat{f}f^​ is random depending on the sampled data, D\mathcal{D}D, which is used to perform the estimation.
  • The expected prediction error of predicting YYY using f^(X)\hat{f}(X)f^​(X) when X=xX = xX=x has been decomposed into two errors:
    • The reducible error, which is the expected squared error loss of estimation f(x)f(x)f(x) using f^(x)\hat{f}(x)f^​(x) at a fixed point xxx. The only thing that is random here is D\mathcal{D}D, the data used to obtain f^\hat{f}f^​. (Both fff and xxx are fixed.) We'll often call this reducible error the mean squared error of estimating f(x)f(x)f(x) using f^\hat{f}f^​ at a fixed point xxx. MSE(f(x),f^(x))≜ED[(f(x)−f^(x))2]\text{MSE}\left(f(x), \hat{f}(x)\right) \triangleq \mathbb{E}_{\mathcal{D}} \left[ \left(f(x) - \hat{f}(x) \right)^2 \right]MSE(f(x),f^​(x))≜ED​[(f(x)−f^​(x))2]
    • The irreducible error. This is simply the variance of YYY given that X=xX = xX=x, essentially noise that we do not want to learn. This is also called the Bayes error.

As the name suggests, the reducible error is the error that we have some control over. But how do we control this error?

Bias-Variance Decomposition

After decomposing the expected prediction error into reducible and irreducible error, we can further decompose the reducible error.

The bias of an estimator is given by

bias(θ^)≜E[θ^]−θ.\text{bias}(\hat{\theta}) \triangleq \mathbb{E}\left[\hat{\theta}\right] - \theta.bias(θ^)≜E[θ^]−θ.

And recall the definition of the variance of an estimator:

V(θ^)=var(θ^)≜E[(θ^−E[θ^])2].\mathbb{V}(\hat{\theta}) = \text{var}(\hat{\theta}) \triangleq \mathbb{E}\left [ ( \hat{\theta} -\mathbb{E}\left[\hat{\theta}\right] )^2 \right].V(θ^)=var(θ^)≜E[(θ^−E[θ^])2].

Using this, we further decompose the reducible error (mean squared error) into bias squared and variance.

MSE(g(x),g^(x))=ED[(g(x)−g^(x))2]=(g(x)−E[g^(x)])2⏟bias2(g^(x))+E[(g^(x)−E[g^(x)])2]⏟var(g^(x))\text{MSE}\left(g(x), \hat{g}(x)\right) = \mathbb{E}_{\mathcal{D}} \left[ \left(g(x) - \hat{g}(x) \right)^2 \right] = \underbrace{\left(g(x) - \mathbb{E} \left[ \hat{g}(x) \right] \right)^2}_{\text{bias}^2 \left(\hat{g}(x) \right)} + \underbrace{\mathbb{E} \left[ \left( \hat{g}(x) - \mathbb{E} \left[ \hat{g}(x) \right] \right)^2 \right]}_{\text{var} \left(\hat{g}(x) \right)}MSE(g(x),g^​(x))=ED​[(g(x)−g^​(x))2]=bias2(g^​(x))(g(x)−E[g^​(x)])2​​+var(g^​(x))E[(g^​(x)−E[g^​(x)])2]​​

(An aside)

This is actually a commonly explored fact in estimation theory, and it is geenrally true. But we have stated this decomposition here specifically for estimation of some regression function ggg using g^\hat{g}g^​ at some point xxx.

MSE(g(x),g^(x))=bias2(g^(x))+var(g^(x))\text{MSE}\left(g(x), \hat{g}(x)\right) = \text{bias}^2 \left(\hat{g}(x) \right) + \text{var} \left(\hat{g}(x) \right)MSE(g(x),g^​(x))=bias2(g^​(x))+var(g^​(x))

In a perfect world, we would be able to find some f^\hat{f}f^​ which is unbiased, that is bias(f^(x))=0\text{bias}\left(\hat{f}(x) \right) = 0bias(f^​(x))=0, which also has low variance. In practice, this isn't always possible.

It turns out, there is a bias-variance tradeoff. That is, often, the more bias in our estimation, the lesser the variance. Similarly, less variance is often accompanied by more bias. Complex models tend to be unbiased, but highly variable. Simple models are often extremely biased, but have low variance.

In the context of regression, models are biased when:

  • Parametric: The form of the model does not incorporate all the necessary variables, or the form of the relationship is too simple. For example, a parametric model assumes a linear relationship, but the true relationship is quadratic.
  • Non-parametric: The model provides too much smoothing.

In the context of regression, models are variable when:

  • Parametric: The form of the model incorporates too many variables, or the form of the relationship is too complex. For example, a parametric model assumes a cubic relationship, but the true relationship is linear.
  • Non-parametric: The model does not provide enough smoothing. It is very, "wiggly."

So for us, to select a model that appropriately balances the tradeoff between bias and variance, and thus minimizes the reducible error, we need to select a model of the appropriate complexity for the data.

Let's explore this relationship in greater detail.

The three plots show three examples of the bias-variance tradeoff. In the left panel, the variance influences the expected prediction error more than the bias. In the right panel, the opposite is true. The middle panel is somewhat neutral. In all cases, the difference between the Bayes error (the horizontal dashed grey line) and the expected prediction error (the solid black curve) is exactly the mean squared error, which is the sum of the squared bias (blue curve) and variance (orange curve). To summarize, if we assume that irreducible error can be written as

V[Y∣X=x]=σ2\mathbb{V}[Y \mid X = x] = \sigma ^ 2V[Y∣X=x]=σ2

then we can write the full decomposition of the expected prediction error of predicting YYY using g^\hat{g}g^​ when X=xX = xX=x as

EPE(Y,f^(x))=bias2(g^(x))+var(g^(x))⏟reducible error+σ2.\text{EPE}\left(Y, \hat{f}(x)\right) = \underbrace{\text{bias}^2\left(\hat{g}(x)\right) + \text{var}\left(\hat{g}(x)\right)}_\textrm{reducible error} + \sigma^2.EPE(Y,f^​(x))=reducible errorbias2(g^​(x))+var(g^​(x))​​+σ2.

As model complexity increases, bias decreases, while variance increases. By understanding the tradeoff between bias and variance, we can manipulate model complexity to find a model that well predict well on unseen observations.

Simulation

We will illustrate these decompositions---most importantly the bias-variance tradeoff---through simulation. Suppose we would like to train a model to learn the function f(x)=x2f(x) = x^2f(x)=x2.

f = function(x) {
  x ^ 2
}

More specifically, we'd like to predict an observation, YYY, given that X=xX = xX=x by using g^(x)\hat{g}(x)g^​(x) where

E[Y∣X=x]=f(x)=x2\mathbb{E}[Y \mid X = x] = f(x) = x^2E[Y∣X=x]=f(x)=x2

and

V[Y∣X=x]=σ2.\mathbb{V}[Y \mid X = x] = \sigma ^ 2.V[Y∣X=x]=σ2.

Alternatively, we could write this as

Y=f(X)+ϵY = f(X) + \epsilonY=f(X)+ϵ

where E[ϵ]=0\mathbb{E}[\epsilon] = 0E[ϵ]=0 and V[ϵ]=σ2\mathbb{V}[\epsilon] = \sigma ^ 2V[ϵ]=σ2. In this formulation, we call f(X)f(X)f(X) the signal and ϵ\epsilonϵ the noise.

To carry out a concrete simulation example, we need to fully specify the data generating process. We do so with the following R code.

get_sim_data = function(f, sample_size = 100) {
  x = runif(n = sample_size, min = 0, max = 1)
  y = rnorm(n = sample_size, mean = f(x), sd = 0.3)
  data.frame(x, y)
}

Also note that if you prefer to think of this situation using the Y=f(X)+ϵY = f(X) + \epsilonY=f(X)+ϵ formulation, the following code represents the same data generating process.

get_sim_data = function(f, sample_size = 100) {
  x = runif(n = sample_size, min = 0, max = 1)
  eps = rnorm(n = sample_size, mean = 0, sd = 0.75)
  y = f(x) + eps
  data.frame(x, y)
}

To completely specify the data generating process, we have made more model assumptions than simply E[Y∣X=x]=x2\mathbb{E}[Y \mid X = x] = x^2E[Y∣X=x]=x2 and V[Y∣X=x]=σ2\mathbb{V}[Y \mid X = x] = \sigma ^ 2V[Y∣X=x]=σ2. In particular,

  • The xix_ixi​ in D\mathcal{D}D are sampled from a uniform distribution over [0,1][0, 1][0,1].
  • The xix_ixi​ and ϵ\epsilonϵ are independent.
  • The yiy_iyi​ in D\mathcal{D}D are sampled from the conditional normal distribution.

Y∣X∼N(f(x),σ2)Y \mid X \sim N(f(x), \sigma^2)Y∣X∼N(f(x),σ2)

Using this setup, we will generate datasets, D\mathcal{D}D, with a sample size n=100n = 100n=100 and fit four models.

predict(fit0, x)=f^0(x)=β^0predict(fit1, x)=f^1(x)=β^0+β^1xpredict(fit2, x)=f^2(x)=β^0+β^1x+β^2x2predict(fit9, x)=f^9(x)=β^0+β^1x+β^2x2+…+β^9x9\begin{aligned} \texttt{predict(fit0, x)} &= \hat{f}_0(x) = \hat{\beta}_0\\ \texttt{predict(fit1, x)} &= \hat{f}_1(x) = \hat{\beta}_0 + \hat{\beta}_1 x \\ \texttt{predict(fit2, x)} &= \hat{f}_2(x) = \hat{\beta}_0 + \hat{\beta}_1 x + \hat{\beta}_2 x^2 \\ \texttt{predict(fit9, x)} &= \hat{f}_9(x) = \hat{\beta}_0 + \hat{\beta}_1 x + \hat{\beta}_2 x^2 + \ldots + \hat{\beta}_9 x^9 \end{aligned}predict(fit0, x)predict(fit1, x)predict(fit2, x)predict(fit9, x)​=f^​0​(x)=β^​0​=f^​1​(x)=β^​0​+β^​1​x=f^​2​(x)=β^​0​+β^​1​x+β^​2​x2=f^​9​(x)=β^​0​+β^​1​x+β^​2​x2+…+β^​9​x9​

These plots start to highlight the difference between the bias and variance of these two models. The zero predictor model is clearly wrong, that is, biased. However, it's nearly the same for each of the datasets---it has very low variance.

While the ninth degree model doesn't appear to be correct for any of these three simulations, we'll see that on average it is, and thus is performing unbiased estimation. These plots do however clearly illustrate that the ninth degree polynomial is extremely variable. Each dataset results in a very different fitted model. Correct on average isn't the only goal you should be after. In practice, we often only have a single dataset but wish to extrapolate our conclusions outside that dataset to the world at large. This is why we'd also like our models to exhibit low variance.

A number of things to notice here:

  • We use squared bias in this table. Since bias can be positive or negative, squared bias is more useful for observing the trend as complexity increases.
  • The squared bias trend which we see here is decreasing as complexity increases, which we expect to see in general.
  • The exact opposite is true of variance. As model complexity increases, variance increases.
  • The mean squared error, which is a function of the bias and variance, decreases, then increases. This is a result of the bias-variance tradeoff. We can decrease bias, by increasing variance. Or, we can decrease variance by increasing bias. By striking the correct balance, we can find a good mean squared error!

A note: we can check for these trends with the diff() function in R.

all(diff(bias ^ 2) < 0)
all(diff(variance) > 0)
diff(mse) < 0

The models with polynomial degrees 2 and 9 are both essentially unbiased. We see some bias here as a result of using simulation. If we increased the number of simulations, we would see both biases go down. Since they are both unbiased, the model with degree 2 outperforms the model with degree 9 due to its smaller variance.

Models with degree 0 and 1 are biased because they assume the wrong form of the regression function. While the degree 9 model does this as well, it does include all the necessary polynomial degrees.

g^9(x)=β^0+β^1x+β^2x2+…+β^9x9\hat{g}_9(x) = \hat{\beta}_0 + \hat{\beta}_1 x + \hat{\beta}_2 x^2 + \ldots + \hat{\beta}_9 x^9g^​9​(x)=β^​0​+β^​1​x+β^​2​x2+…+β^​9​x9

Then, since least squares estimation is unbiased, importantly,

E[β^d]=βd=0\mathbb{E}[\hat{\beta}_d] = \beta_d = 0E[β^​d​]=βd​=0

for d=3,4,…9d = 3, 4, \ldots 9d=3,4,…9, we have

E[g^9(x)]=β0+β1x+β2x2\mathbb{E}\left[\hat{g}_9(x)\right] = \beta_0 + \beta_1 x + \beta_2 x^2E[g^​9​(x)]=β0​+β1​x+β2​x2

Now we can finally verify the bias-variance decomposition.

bias ^ 2 + variance == mse

But wait, this says it isn't true, except for the degree 9 model? It turns out, this is simply a computational issue. If we allow for some very small error tolerance, we see that the bias-variance decomposition is indeed true for predictions from these for models.

Estimating Expected Prediction Error

Assuming

V[Y∣X=x]=σ2.\mathbb{V}[Y \mid X = x] = \sigma ^ 2.V[Y∣X=x]=σ2.

we have

Lastly, we note that if

D=Dtrn∪Dtst=(xi,yi)∈Rp×R, i=1,2,…n\mathcal{D} = \mathcal{D}_{\texttt{trn}} \cup \mathcal{D}_{\texttt{tst}} = (x_i, y_i) \in \mathbb{R}^p \times \mathbb{R}, \ i = 1, 2, \ldots nD=Dtrn​∪Dtst​=(xi​,yi​)∈Rp×R, i=1,2,…n

where

Dtrn=(xi,yi)∈Rp×R, i∈trn\mathcal{D}_{\texttt{trn}} = (x_i, y_i) \in \mathbb{R}^p \times \mathbb{R}, \ i \in \texttt{trn}Dtrn​=(xi​,yi​)∈Rp×R, i∈trn

and

Dtst=(xi,yi)∈Rp×R, i∈tst\mathcal{D}_{\texttt{tst}} = (x_i, y_i) \in \mathbb{R}^p \times \mathbb{R}, \ i \in \texttt{tst}Dtst​=(xi​,yi​)∈Rp×R, i∈tst
